VARIABILI E COSTANTI

In c possiamo creare delle variabili in cui memorizzare alcune informazioni
int(Numeri interi), float(numeri decimali) e char(singoli caratteri).

INPUT DA TASTIERA

Le variabili possono essere aggiornate non solo assegnandoci un valore a mano, ma anche chiedendo un valore in input all'utente, questo grazie al comando scanf(), ricordando di specificare il tipo di dato da inserire.
scanf("%d", &NomeVariabile); //Chido in input un numero intero

IF

Se si vogliono eseguire di semplici controlli su delle variabili è possibile usare il costrutto if, aggiungendo else alla fine possiamo gestire anche il caso che la condizione sia falsa.
if(varibile>3){
printf("Numero maggiore di 3");
}else{
printf("numero minore o uguale a 3");
}

VETTORI

Se dobbiamo memorizzare una serie di variabili tutte dello stesso tipo possiamo usare un array, ovvero una catena di variabili ognuna identificata da un indice che parte da 0.
In C per dichiarare un vettore è necessario definire anche la sua grandezza e non sarà più possibile modificarla
int vettore=[10];
vettore[3]=3;

FOR & WHILE

Può succedere che sia necessario eseguire un azione o una serie di operazioni più volte di seguito(esempio: riempire un vettore), in questo vaso si possono usare i cogli while e for.
Il ciclo For si utilizza nel caso io sappia quante volte devo eseguire un azione, il while invece esegue un azione fino a quando la condizione non diventa falsa.
int i=0;
for(i=0; i< 10; i++){
printf(i);
}

while(i==10){
printf(i);
i++;}

SWITCH

Nel caso si necessiti di controllare se una variabile si uguale a una serie di valori, e i base al valore esegiore azioni differenti si può optare per uno switch al posto che una serie di if
switch (varibile){
case opzione1:
printf(1);
break;
case opzione2:
printf(2);
break;
case opzione3:
printf(3);
break;
}

PROCESSI

In C possiamo creare programmi che non generino un singolo processo quando vengono eseguiti, ma che ne generino di più, in modo da svolgere più azioni contemporaneamente, per farlo è necessario importare la libreria unistd.h
#include < unistd.h >
int main(){
pid_t ChiSono; // varibile di tipo pid_t che serve a sapere che genera il processo
ChiSono=fork();
//la funzione fork() genera 1 processo, ritorna un numero < 0 se genera 1 errore, 0 se ci si trova nel processo figlio e > 0 se ci si trova nel processo padre
return 0;
}
getpid() //ritorna il pid del processo
getppid() //ritorna il pid del processo padre, nel caso si esegua il comando all'inizio del programma, e non in un processo "figlio", ritornarà il pid del processo del tarminale da cui è stato eseguito

MEMORIA CONDIVISA

Un limite del lavorare con i processi in C è le variabili non sono condivise, ma ogni processo crea una copia dell'intero programma, salvando le variabili in aree differenti di memoria.
Per questo se si vuole passare dei valori tra un processo ed un altro disogna creare una memoria condivisa.
#include < sys/ipc.h >
#include < sys/shm.h >
#include < sys/types.h >
#define SHM_SIZE 1024 /* dimensione della memoria = 1KB */
int main(){
key_t chiave; //varibile che contiene la chiave per accedere alla memoria condivisa
int shmid;//id della memoria condivisa
chiave=1234;
shmid=shmget(chiave, SHM_SIZE, 0666 | IPC_CREAT );
//shmid=shmget(key, dimensione, permessi di accesso e flag di creazione)
p=(int *)shmat(id, NULL, SHM_RDONLY);// per leggere la memoria, p è un puntatore ad area di memoria di tipo intero.
p=(int *)shmat(id, NULL, 0);//per scrivere cambio semplicemente il flag
shmctl(shmid, IPC_RMID, NULL);
//shmctl(shmid, cmd, puntatore ad una struttura);
retunrn 0;
}

SEMAFORI

Quando si ha una memoria condivisa con tati processi che leggono e scrivono, si può finire con più processi che svolgono la stessa azione contemporaneamente, nel caso della lettura non si generano problemi, ma se tanti processi scrivono contemporaneamente nell'area di memoria protrebbero generarsi neìumerosi problemi.
Per ovviare a questi problemi si possono inserire dei semafori per impedire l'accesso alle risorse.
#include < sys/ipc.h >
#include < sys/sem.h >
id=semget(key_t semKey, int nsems, int options);
//creazione di un semafori shmget(chiave del semaforo, numero di semafori da creare, permessi)
semctl(id, int sem_num, GETVAL);
//per ottere il valore del semafori passo alla funzione semctll'id, l'indice del semaforo da creare e dico che voglio ottenere il valore con GETVAL
semctl(id, int sem_num, SETVAL, ValoreDaSettare);
//la sentassi è uguale a quella del per leggere il valore, solo che cambio il flag e passo anche il volore da settare.
semctl(id, int sem_num, IPC_RMID);
// con il flag IPC_RMID indico che voglio distruggere il gruppo di semafori
