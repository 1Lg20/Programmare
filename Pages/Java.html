VARIABILI E ARRAY

Per salvare dei valori possiamo usare delle variabili, o volendo possiamo usare degli array nel caso abbiamo più valori dello stesso tipo.
Bisogna ricordare che una variabile deve essere definita privata o pubblica (volendo anche accessibile per ereditarietà, quindi solo le classi che ereditano da quella classe, o a tutto il package), in base a chi può accedere a quella risorsa
private String Targa;
private int max;
int[] myArray = new int[5];

IF

Per eseguire un controllo semplice su delle variabili posso usare in semplice if con una condizione ed un else che viene eseguito nel caso la condizione sia falsa
if(variabile==0){
System.out.println(“0”);
}else{
System.out.println(“Non 0”);
}

WHILE E FOR

Per eseguire una stessa azione più volte di seguito(Es:Stampare gli elementi di un Array) possiamo usare un ciclo while o un ciclo for, la differenza tra i 2 è che i for gira per un numero finito di volte, mentre il while fino a quando la condizione non diventa falsa
while(condizione){
System.out.println(“Sono nel while”);
}

for(int i=0; i < 10; i++){
System.out.println(“Sono nel for”);
}

CLASSI

Java è un linguaggio che lavora per classi, vuol dire che posso creare istanze di queste classi che svolgono la stessa funzione, per esempio se nel mio programma devo gestire un parcheggio, posso creare una classe automobile, e poi creare un'istanza di questa classe per ogni automobile.
Rispetto ad una variabile semplice una classe permette di avere numerosi attributi, dei campi che devo valorizzare quando creo un'istanza di quella classe.
Ad una classe posso passare un parametro o più proprio come farei con una funzione, quindi le variabili primitive(numeri interi, float, boolean ed anche le stringhe) sono passate per valore, quindi salvando il valore in una nuova area di memoria, mentre tutto il resto(Array, classi, ecc.) sono passati per parametro, quindi una modifica all’interno della classe sarà visibile a tutto il programma
//Main.java
Macchina auto1=new Macchina("aa00bb")

//File Macchina.java
public class Macchina{
private String Targa;
public Macchina(String targa) {
this.Targa = targa;
}
}

SINCRONIZZAZIONE

Ovviamente non possiamo far accedere più thread ad una stessa risorsa, fino a quando si deve solo fare una lettura di una variabile o qualcosa di simile non ci sono problemi, ma nel momento in cui bisogna andare a scrivere il quella variabile/file/… potrebbero generarsi problemi visto che più thread provano a scrivere, sovrascrivendo ogni volta il precedente.
Per risolvere il problema possiamo sincronizzare i vari thread in modo che 1(ò più di uno) alla volta usino la risorsa.
Per farlo è sufficiente usare wait e notifyall per bloccare la risorsa o per dire che abbiamo terminato l’utilizzo della risorsa, il tutto dentro una funzione sincronizzata.
public synchronized LocalTime Entra(String Targa) {
try {
wait();
} catch (InterruptedException e) {
e.printStackTrace();
